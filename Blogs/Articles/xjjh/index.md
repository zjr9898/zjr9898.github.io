## 信竞邪神：嘉豪的离谱算法AK之路（逆天改命版）

### 零、序章：从Hello World开始崩溃

嘉豪第一次写代码，运行printf("Hello World");，终端输出的是"恭喜发财"。

他把电脑拆开，发现CPU里住着一只蟑螂。他把蟑螂请出去，从此他的代码不需要编译——蟑螂留下的轨迹就是最优指令集。

### 一、冒泡排序优化版·究极体

省赛求逆序对，嘉豪的代码：

```cpp
// 时间复杂度：O(1) 因为只循环一次
int ans = 0;
for(int i=0; i<1; i++){  // 信仰：一次就够
    if(a[0] > a[1]) ans++;
    if(a[1] > a[2]) ans++;
    // ... 省略99997行手写判断
}
```

1.5MB的源代码，全部手写展开。编译后的exe文件大小超过100MB，但运行只需要0.000001秒——因为操作系统直接把这段代码识别成了硬件病毒，在CPU微码层面执行。

评测机看到这段代码后当场蓝屏，重启后自动判定AC。

### 二、蒙特卡洛树搜索·因果律修正版

省选最短路，嘉豪的random_neighbor函数升级了：

```cpp
int random_neighbor(int u){
    // 读取宇宙微波背景辐射作为随机种子
    FILE* cmb = fopen("/dev/宇宙", "r");
    unsigned long long seed;
    fread(&seed, sizeof(seed), 1, cmb);
    fclose(cmb);
    
    // 使用曼德博集合迭代10000次
    double r = 0;
    for(int i=0; i<10000; i++) r = r*r + seed%100/100.0;
    
    // 关键：根据输出结果反向修正随机概率
    if(本次路径将会是最优解){
        return 正确邻居;
    } else {
        return 另一个邻居;
    }
}
```

问题在于C++标准库并没有/dev/宇宙这个设备。但嘉豪自己写了一个内核模块，通过监听宇宙射线的能级变化来生成真随机数。他管这叫因果律编程——不是算法找到最优解，而是最优解找到算法。

赛后有人在嘉豪的U盘里发现了一个名为时间机器驱动程序.sys的文件。

### 三、goto地狱飞升·时间线分支版

NOI压轴题，嘉豪的goto升级为多重宇宙goto：

```cpp
// 不是goto到代码行，而是goto到平行宇宙的代码行
multiverse_goto label_start, label_end, label_inf;

// 在每个宇宙中同时执行不同的goto分支
#pragma omp parallel num_threads(INFINITY)
{
    int universe_id = get_universe_id();
    if(universe_id % 2 == 0){
        goto up_u;  // 这个宇宙的嘉豪往上跳
    } else {
        goto up_v;  // 这个宇宙的嘉豪往下跳
    }
}
// 当至少一个宇宙返回答案时，坍缩所有宇宙
return collapse_wavefunction(ans);
```

编译器报错："未定义标识符multiverse_goto"

嘉豪淡定地发了一个补丁给GCC维护组，把multiverse_goto加入了C++26标准草案。邮件往来记录显示，Linus Torvalds回复了一句："This is not crazy, this is art."

### 四、量子波动速读·现场改题版

ICPC WF，遇到不可判定问题。嘉豪没写随机数，他做了一件更逆天的事：

他走到评委面前说："这题数据太弱了，我帮你们加强一下。"然后现场把输入数据的约束从n ≤ 10^5改成了n ≤ 10，再把题目要求从"判断停机"改成了"判断1+1是否等于2"。

评委还没反应过来，嘉豪已经坐回座位，敲下：

```cpp
#include <iostream>
int main(){
    std::cout << 2 << std::endl;
    return 0;
}
```

AC。耗时0ms，内存4KB。

ICPC官方事后发布声明："经查，嘉豪没有物理接触评测系统。他只是在评委的大脑里执行了一次社会工程学攻击，改变了他们对题目的记忆。"

嘉豪对此的回应："这叫社交工程复杂度O(1)。"

### 五、终极AK术：把自己编译进评测机

IOI最后一题，嘉豪没有写任何算法。他写了一个bootloader，把自己电脑的操作系统装进了评测机的UEFI固件里。

然后他通过这个后门，直接读取了所有测试点的正确答案，用显影剂写在三文鱼上，再通过OCR扫描回代码里。

```cpp
// 请评委老师品尝一下这条三文鱼
// 答案藏在鱼肉的脂肪纹路里
```

评委吃了三文鱼，突然顿悟：“答案是42！”然后手动给嘉豪加了分。

整个过程不需要编译，不需要运行，复杂度为O(吃)。

### 六、邪神飞升·超越竞赛

嘉豪AK了所有竞赛之后，出题人们联合开会，决定专门为嘉豪设计一道无法被邪修通过的题：一个依赖外部输入真随机数的交互题，每次运行时宇宙熵值都不相同。

嘉豪看了题，沉思三秒，然后在代码开头写：

```cpp
#define true (rand() % 2)
```

然后他写了一个死循环：

```cpp
while(true){
    // 因为true有一半概率为false
    // 所以循环会恰好执行一次后break
    solve_problem();
    break;
}
```

AC。

评委崩溃了：“你这不是编程，你这是重新定义了真值表！”

嘉豪在颁奖典礼上说了一番震动整个计算机界的话：

“你们觉得布尔值只有true和false，那是因为你们的逻辑是二值的。在我的逻辑里，true有37.5%的概率是false，剩余概率是‘评委觉得对就行’。

你们说数组越界是UB，我管那叫‘内存探索权’。

你们说浮点误差不能忍，我说小数点后第114514位才是真正的答案。

你们说复杂度不能O(∞)，我说∞/2 = ∞，所以常数优化一下就是O(∞/2) = O(∞)，没区别。

计算机科学不是发现规律，是发明规律。而我是发明规律的人。”

从此，嘉豪不再参加竞赛。他去做了编译器——写了一个能把任何代码编译成“直接返回AC”的神级编译器，取名gx豪。

评测机们终于安息了。
